> **版本**: 1.4.0 | **更新日期**: 2026-06-05 | **文档版本**: 0.1

## 1. 简介

AgentHub（代号 Loom）是一个**多智能体协作平台**，提供 AI Agent 的创建、配置、编排、监控和成果物管理的完整生命周期支持。平台支持单 Agent 对话与多 Agent 群组协作，通过 Orchestrator 调度引擎实现任务的自动分解与分发。

### 1.1 目标受众

| 角色 | 使用场景 |
|------|----------|
| **平台管理员** | 部署维护、模型供应商配置、安全策略管理、用户管理 |
| **工作空间管理员** | Agent 创建与配置、Skill/Tool 绑定、成员权限管理 |
| **普通用户** | 与 Agent 对话、创建群组协作、管理 Artifact 成果物 |
| **开发者** | API 集成、自定义 Skill 开发、MCP Server 接入、ACP 适配器开发 |

### 1.2 核心能力

- **多 Agent 管理**: 支持 ReAct、Plan-Execute、Orchestrator 三种 Agent 类型
- **SSE 流式对话**: 实时推送文本增量、工具调用、任务计划等事件
- **群组协作**: Orchestrator 自动分解任务并分发给多个 Agent 并行/串行执行
- **Skill/Tool 生态**: 可扩展的技能与工具绑定机制，支持 MCP 协议接入
- **Artifact 管理**: Agent 产出的代码、网页、文档等成果物的版本管理与一键部署
- **多工作空间**: 租户级资源隔离与角色权限控制
- **多渠道集成**: 支持 Web、钉钉、飞书、Slack、Discord 等渠道接入

### 1.3 技术栈

| 层级 | 技术 |
|------|------|
| **前端** | Vue 3 (Composition API) + Pinia + Vue Router + Naive UI + Vite |
| **后端** | Java 21 + Spring Boot 3.5 + Spring AI 1.1.6 + MyBatis Plus 3.5 |
| **数据库** | 开发环境 H2，生产环境 MySQL 8.0，Flyway 迁移管理 |
| **通信** | SSE 流式推送 + WebSocket + REST API |
| **认证** | JWT + PAT 双令牌机制，BCrypt 密码加密 |
| **构建** | Maven 多模块（后端）+ npm/Vite（前端） |
| **部署** | Docker 多阶段构建，支持 Docker Compose 编排 |

## 2. 快速开始

### 2.1 环境要求

| 软件 | 版本要求 |
|------|----------|
| JDK | 21+ |
| Maven | 3.9+ |
| Node.js | 22+ |
| npm | 10+ |
| MySQL | 8.0+（生产环境） |
| Docker | 可选，用于容器化部署 |

### 2.2 启动后端（开发模式）

后端使用 H2 文件数据库，无需额外安装数据库：

```bash
cd mateclaw-dev
mvn spring-boot:run -pl mateclaw-server
```

后端启动后监听 `http://localhost:18088`，API 基路径为 `/api/v1/`。

**Swagger UI**: `http://localhost:18088/swagger-ui.html`
**H2 Console**: `http://localhost:18088/h2-console`（需设置环境变量 `H2_CONSOLE_ENABLED=true`）

### 2.3 启动前端（开发模式）

```bash
cd AIagent_frontend
npm install
npm run dev
```

前端开发服务器监听 `http://localhost:3000`，Vite 自动代理 `/api` 请求到 `http://127.0.0.1:18088`。

### 2.4 使用 MySQL（生产环境）

```bash
# 设置环境变量
export DB_HOST=localhost
export DB_PORT=3306
export DB_NAME=mateclaw
export DB_USERNAME=root
export DB_PASSWORD=your_password

# 以 MySQL 模式启动
cd mateclaw-dev
mvn spring-boot:run -pl mateclaw-server -Dspring-boot.run.profiles=mysql
```

### 2.5 Docker 部署

```bash
cd mateclaw-dev/mateclaw-server
docker build -t mateclaw-server .
docker run -p 18088:18088 \
  -e DB_HOST=your_mysql_host \
  -e DB_USERNAME=root \
  -e DB_PASSWORD=your_password \
  mateclaw-server
```

### 2.6 初始化系统

首次启动后，访问 `http://localhost:3000` 会自动跳转到初始化页面。完成初始设置后，使用管理员账号登录即可使用。

## 4. 核心功能模块

### 4.1 Agent 管理

Agent 是平台的核心抽象，代表一个可对话的 AI 智能体。每个 Agent 拥有独立的系统提示词、模型配置、能力标签和绑定的 Skill/Tool。

**Agent 类型**:

| 类型 | 标识 | 说明 |
|------|------|------|
| ReAct | `react` | 标准推理-行动模式 Agent，适合大多数对话场景 |
| Plan-Execute | `plan_execute` | 先制定计划再逐步执行，适合复杂多步任务 |
| Orchestrator | `orchestrator` | 多 Agent 协调者，负责任务分解与分派 |

**Agent 状态**:

| 状态 | 说明 |
|------|------|
| `AVAILABLE` | 就绪，可接受新任务 |
| `BUSY` | 正在处理任务 |
| `OFFLINE` | 已禁用或不可达 |

#### 创建 Agent (API)

```bash
curl -X POST http://localhost:18088/api/v1/agents \
  -H "Authorization: Bearer <token>" \
  -H "X-Workspace-Id: 1" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Claude Code",
    "description": "Expert coding agent powered by Claude",
    "agentType": "react",
    "systemPrompt": "You are an expert software engineer...",
    "modelName": "claude-sonnet-4-6",
    "enabled": true,
    "capabilityTags": ["coding", "debugging"],
    "isPublic": true
  }'
```

#### 绑定 Skill 到 Agent

```bash
curl -X PUT http://localhost:18088/api/v1/agents/5/skills \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '[1, 2, 3]'
```

### 4.2 SSE 流式对话

平台的核心交互方式是基于 SSE（Server-Sent Events）的流式对话。

#### 发起流式对话

```javascript
// 前端使用 fetch 发起 SSE 请求
const response = await fetch('/api/v1/chat/stream', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`,
    'X-Workspace-Id': workspaceId
  },
  body: JSON.stringify({
    agentId: 5,
    message: 'Create a React component for a user profile card',
    conversationId: null,
    thinkingLevel: 'medium'
  })
})

const reader = response.body.getReader()
// 解析 SSE 事件流...
```

#### SSE 事件类型

| 事件 | 说明 |
|------|------|
| `text` | 增量文本输出 |
| `tool_call` | Agent 调用工具 |
| `tool_result` | 工具执行结果 |
| `orchestrator_plan` | 群组场景：Orchestrator 任务分解计划 |
| `delegation_progress` | 群组场景：子 Agent 执行进度 |
| `artifact_preview` | Agent 产出 Artifact 预览 |
| `done` | 本轮对话完成 |
| `error` | 发生错误 |

#### SSE 重连机制

客户端可通过 `Last-Event-Id` 头实现断线重连：

```javascript
// 重连请求
fetch('/api/v1/chat/stream', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    conversationId: 'existing-conv-id',
    reconnect: true,
    lastEventId: 'event-42'
  })
})
```

服务端通过 `ChatStreamTracker` 缓存最近 16000 条事件，重连时先回放缓存事件再继续推送新事件。

### 4.3 群组协作与 Orchestrator

群组对话允许多个 Agent 在 Orchestrator 的协调下协作完成任务。

#### 创建群组对话

```bash
curl -X POST http://localhost:18088/api/v1/conversations/group \
  -H "Authorization: Bearer <token>" \
  -H "X-Workspace-Id: 1" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Website Redesign Team",
    "orchestratorAgentId": 3,
    "agentIds": [5, 7, 8],
    "schedulingMode": "auto",
    "failurePolicy": "fail_tolerant",
    "maxParallelTasks": 4
  }'
```

**调度模式**:

| 模式 | 说明 |
|------|------|
| `auto` | Orchestrator 自动根据能力匹配分配任务 |
| `manual` | 用户手动 @mention Agent 分配子任务 |

**失败策略**:

| 策略 | 说明 |
|------|------|
| `fail_fast` | 任一子任务失败则停止全部 |
| `fail_tolerant` | 继续执行其他子任务，最后报告失败项 |

#### 监控任务执行

```bash
# 查看任务详情
curl http://localhost:18088/api/v1/orchestrator/tasks/101 \
  -H "Authorization: Bearer <token>"

# 查看子任务分配
curl http://localhost:18088/api/v1/orchestrator/tasks/101/assignments \
  -H "Authorization: Bearer <token>"

# 重试失败子任务
curl -X POST http://localhost:18088/api/v1/orchestrator/tasks/101/retry \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"assignmentIds": [201]}'
```

### 4.4 Artifact 成果物管理

Agent 生成的网页、代码、文档等产出物作为 Artifact 进行版本管理。

**Artifact 类型**: `html`, `code`, `markdown`, `pdf`, `ppt`, `image`, `other`

#### 版本历史与 Diff

```bash
# 查看版本历史
curl http://localhost:18088/api/v1/artifacts/1/versions \
  -H "Authorization: Bearer <token>"

# 版本对比
curl "http://localhost:18088/api/v1/artifacts/1/versions/diff?from=1&to=3" \
  -H "Authorization: Bearer <token>"

# 回滚到历史版本
curl -X POST http://localhost:18088/api/v1/artifacts/1/versions/2/restore \
  -H "Authorization: Bearer <token>"
```

#### 一键部署

```bash
# 发起部署
curl -X POST http://localhost:18088/api/v1/artifacts/1/deploy \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"deployTarget": "vercel", "versionId": 10}'

# 查看部署状态
curl http://localhost:18088/api/v1/artifacts/1/deploy/status \
  -H "Authorization: Bearer <token>"
```

### 4.5 Skill 系统

Skill 是 Agent 的可插拔能力模块，包含系统提示词片段和可选的脚本文件。

```bash
# 列出技能
curl "http://localhost:18088/api/v1/skills?keyword=coding&enabled=true" \
  -H "Authorization: Bearer <token>" \
  -H "X-Workspace-Id: 1"

# 预览技能的渲染提示词
curl http://localhost:18088/api/v1/skills/1/prompt-preview \
  -H "Authorization: Bearer <token>" \
  -H "X-Workspace-Id: 1"
```

### 4.6 MCP 集成

支持 Model Context Protocol，可接入外部工具服务器。

```bash
# 注册 MCP Server
curl -X POST http://localhost:18088/api/v1/mcp/servers \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Filesystem Server",
    "transport": "stdio",
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"],
    "enabled": true
  }'

# 测试连接
curl -X POST http://localhost:18088/api/v1/mcp/servers/1/test \
  -H "Authorization: Bearer <token>"

# 发现工具
curl -X POST http://localhost:18088/api/v1/mcp/servers/1/discover \
  -H "Authorization: Bearer <token>"
```

### 4.7 Memory 系统

平台提供 Agent 记忆系统，支持从对话中提取事实、梦境巩固和记忆涌现。

```bash
# 触发记忆涌现
curl -X POST http://localhost:18088/api/v1/memory/5/emergence \
  -H "Authorization: Bearer <token>"

# 查看已存储的事实
curl http://localhost:18088/api/v1/memory/5/facts?page=1&size=20 \
  -H "Authorization: Bearer <token>"

# 对事实提供反馈
curl -X POST http://localhost:18088/api/v1/memory/5/facts/10/feedback \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"feedback": "accurate"}'
```

## 5. API 接口参考

**Base URL**: `http://localhost:18088/api/v1/`

**统一响应格式**:

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

**分页格式**:

```json
{
  "code": 200,
  "data": {
    "records": [],
    "total": 150,
    "current": 1,
    "size": 20,
```

## 7. 开发指南

### 7.1 后端开发规范

#### 分层架构（强制）

```
Controller → Service → Mapper
```

- **Controller**：只负责请求解析、参数校验、调用 Service、返回 `R<T>`
- **Service**：所有业务逻辑，写操作加 `@Transactional`
- **Mapper**：在 `vip.mate.domain.**.repository` 包下，继承 `BaseMapper<T>`

#### 依赖注入

始终使用 Lombok `@RequiredArgsConstructor` 构造器注入，禁止字段注入：

```java
// ✅ 正确
@RequiredArgsConstructor
public class AgentServiceImpl implements AgentService {
    private final AgentMapper agentMapper;
}

// ❌ 错误
@Autowired
private AgentMapper agentMapper;
```

#### API 设计

- 基路径 `/api/v1/`，资源用复数名词
- 正确: `/api/v1/agents/{id}` / 错误: `/api/v1/getAgent`
- 统一使用 `R<T>` 返回，异常由 `RHttpStatusAdvice` 全局处理

#### 数据库迁移

迁移脚本放在 `mateclaw-server/src/main/resources/db/migration/h2/`，命名格式 `V{序号}__{描述}.sql`。H2 使用 MySQL 兼容模式。

### 7.2 前端开发规范

#### 状态管理

- 组件私有状态使用 `ref` / `reactive` / `computed`
- 跨组件共享状态使用 Pinia setup stores
- 服务端状态不要复制到 Pinia，走 API 层

```javascript
// stores/example.js
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useExampleStore = defineStore('example', () => {
  const items = ref([])
  const count = computed(() => items.value.length)

  async function load() {
    const data = await api.getItems()
    items.value = data
  }

  return { items, count, load }
})
```

#### SSE 使用

```javascript
// composables/useSSE.js 提供通用 SSE 能力
const { on, disconnect } = useSSE('/api/v1/chat/stream', { body: requestBody })

on('text', (data) => {
  // 追加文本到消息内容
  appendContent(data.delta)
})

on('done', (data) => {
  // 对话完成
  finishMessage(data.tokenUsage)
})

// 组件卸载时自动断开连接
```

### 7.3 添加新功能

1. **数据库变更**：创建新的 Flyway 迁移脚本（H2 和 MySQL 各一份）
2. **领域模型**：在 `mateclaw-domain` 中添加 Entity、Mapper、Service
3. **API 端点**：在 `mateclaw-server` 中添加 Controller
4. **前端页面**：添加视图组件、路由、Store、API 模块

## 8. 安全模型

### 8.1 认证流程

```
客户端                     服务端
  │                         │
  │  POST /auth/login       │
  │  {username, password}   │
  │────────────────────────→│
  │                         │ 验证密码 (BCrypt)
  │  返回 JWT Token         │ 生成 24h 有效 JWT
  │←────────────────────────│
  │                         │
  │  后续请求携带            │
  │  Authorization: Bearer  │
  │  <jwt_token>            │
  │────────────────────────→│
  │                         │ JwtAuthFilter 验证
  │  若 Token 有效期 < 2h   │
  │  X-New-Token: <new>    │ 自动续期
  │←────────────────────────│
```

### 8.2 角色权限

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| `viewer` | 只读 | 查看对话、Agent、内容 |
| `member` | 读写 | 管理 Agent、对话 |
| `admin` | 管理 | 管理成员、渠道、模型、安全策略 |
| `owner` | 所有者 | admin 权限 + 删除工作空间 |
| `global_admin` | 全局管理员 | 用户管理、运行时控制 |

### 8.3 安全措施

- JWT Token 24 小时过期，支持滑动续期
- PAT（Personal Access Token）使用 bcrypt 哈希存储，明文仅创建时返回一次
- 密码使用 BCrypt 加密存储
- SQL 注入防护：手写 SQL 使用 `#{}`参数化，禁止 `${}` 拼接用户输入
- 所有 API 默认需认证（白名单除外）
- 密钥、Token、数据库密码通过环境变量注入，禁止硬编码

## 9. 常见问题

### 9.1 前端代理不生效

确认 `vite.config.js` 中代理配置正确，后端运行在 `18088` 端口。检查浏览器 Network 面板确认请求路径是否正确。

### 9.2 SSE 连接频繁断开

检查以下配置：
- SSE emitter 超时时间（默认 10 分钟）是否大于 LLM 调用最长响应时间
- 心跳间隔配置是否合理（token 前 2s，流式期间 10s 一次 `: heartbeat` 注释）
- 确认反向代理（如 Nginx）未缓冲 SSE 响应（需关闭 `proxy_buffering`）

### 9.3 数据库迁移失败

确认 Flyway 迁移脚本同时在 `h2/` 和 `mysql/` 两个目录下都有对应版本。H2 环境下需确保 SQL 语法兼容 MySQL 模式。

### 9.4 MyBatis Mapper 扫描不到

所有 Mapper 接口必须位于 `vip.mate.domain.**.repository` 包路径下，否则 `@MapperScan` 不会注册。

### 9.5 Token 过期处理

前端 Axios 拦截器自动检测 401 状态码，清除 Token 并重定向到登录页。同时自动检测 `x-new-token` 响应头，在 Token 剩余有效期不足 2 小时时自动续期。
